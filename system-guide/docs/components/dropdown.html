3가지 드롭다운 타입

타입 1: 선택형 (listbox)

용도: 상태/기준 선택, 필터링/정렬 기준 설정
  • CRUD 200
  • 목록/리스트 150 
<!-- 선택형 드롭다운 -->
<div class="dropdown" data-type="listbox">
  <button type="button" class="dropdown__button" aria-expanded="false" aria-controls="program-type-2">
    <span class="dropdown__text">선택형(listbox) 유형</span>
  </button>
  <ul class="dropdown__list" id="program-type-2" role="listbox" hidden>
    <li class="dropdown__item" role="option">
      <span class="dropdown__item-text">CRUD</span>
      <span class="dropdown__count">200</span>
    </li>
  </ul>
</div>

타입 2: 액션형 (menu)

용도: 기능/액션 실행, 각 옵션이 액션 역할

    <span class="dropdown__text">액션형(menu) 유형</span>
  </button>
  <ul class="dropdown__list" id="row-actions-2" role="menu" hidden>
    <li role="none">
      <button type="button" role="menuitem" class="dropdown__item" tabindex="-1">
        <span class="dropdown__item-text">편집</span>
        <span class="dropdown__count">Ctrl+E</span>
      </button>
    </li>
  </ul>
</div>

타입별 사용 가이드

타입 선택 기준:
  • 선택형 (listbox): 필터링, 정렬, 상태 선택
  • 액션형 (menu): 기능 실행, 작업 수행
  • 디스클로저형 (disclosure): 정보 표시, 도움말, 설명
자동 타입 감지:
  • data-type 속성으로 명시적 지정
  • 없으면 내부 role 속성으로 자동 유추
  • JavaScript 초기화는 동일하게 App.a11y.initDropdown 사용

접근성 가이드

최소 안전 사양 (필수)

중요: 심사에서 "불필요" 지적을 피하기 위해 최소한의 필수 속성만 명시합니다. 아래 속성들은 접근성의 핵심이며 반드시 포함되어야 합니다.

선택형 (listbox)

<!-- 선택형 드롭다운 최소 사양 -->
<button type="button" aria-expanded="false" aria-controls="program-type-3">
  <span class="dropdown__text">프로그램 유형</span>
</button>
<ul id="program-type-3" role="listbox" hidden>
  <li role="option" tabindex="-1">CRUD</li>
  <li role="option" tabindex="-1">목록/리스트</li>
  <li role="option" tabindex="-1" aria-selected="true">조회(R)</li>
</ul>
필수 속성:
  • aria-expanded: 열림/닫힘 상태 전달
  • aria-controls="program-type-3": 버튼이 제어하는 리스트의 ID
  • role="listbox": 선택 가능한 옵션들의 컨테이너
  • role="option": 각 아이템이 선택 가능한 옵션
  • tabindex="-1": 팝업형에서 Tab으로 메뉴를 빠져나가고 ↑/↓로만 이동
  • aria-selected="true": 선택된 항목에만 표기 (나머지는 생략)

액션형 (menu)

<!-- 액션형 드롭다운 최소 사양 -->
<button type="button" aria-expanded="false" aria-controls="row-actions-3">
  작업 선택
</button>
<ul id="row-actions-3" role="menu" hidden>
  <li role="none">
    <button type="button" role="menuitem" tabindex="-1">편집</button>
  </li>
  <li role="none">
    <button type="button" role="menuitem" tabindex="-1" aria-disabled="true">삭제</button>
  </li>
</ul>
필수 속성:
  • aria-expanded: 열림/닫힘 상태 전달
  • aria-controls="row-actions-3": 버튼이 제어하는 메뉴의 ID
  • role="menu": 액션 메뉴임을 명시
  • role="menuitem": 실행 가능한 액션임을 명시
  • tabindex="-1": Tab이 메뉴 밖으로 빠지도록 함
  • aria-disabled="true": 필요할 때만 비활성 항목에 표기
주의: aria-selected는 사용하지 않음 (명령 실행 목록이므로 "선택 상태" 개념 없음)

디스클로저형 (disclosure)

<!-- 디스클로저형 드롭다운 최소 사양 -->
<button type="button" aria-expanded="false" aria-controls="help-panel-3">
  도움말 보기
</button>
<div id="help-panel-3" hidden>
  <!-- 일반 콘텐츠 -->
  <h4>사용 방법</h4>
  <p>이 드롭다운을 사용하여 원하는 옵션을 선택할 수 있습니다.</p>
</div>
필수 속성:
  • aria-expanded: 열림/닫힘 상태 전달
  • aria-controls="help-panel-3": 버튼이 제어하는 패널의 ID
  • hidden: 패널의 가시성 제어
주의: 단순 정보 패널이면 별도 role 불필요. role="region"·aria-labelledby는 중요 랜드마크로 취급할 때만 고려.

권장 사항 (선택)

참고: 아래 속성들은 접근성을 향상시키지만 "필수"는 아닙니다. 심사에서 "불필요"로 지적될 수 있으므로 선택적으로 적용하세요.
권장 속성:
  • aria-haspopup="listbox" | aria-haspopup="menu": 스크린리더가 "팝업 있음"을 선행 안내 (선택적)
  • role="none" (menu에서 li 래퍼): 중첩 역할 충돌 예방 (선택적)
  • role="region" + aria-labelledby (디스클로저): 실질적 랜드마크 필요 시에만 (선택적)
디스클로저 특별 주의사항:
  • 디스클로저는 기본적으로 별도 role 불필요하며, aria-haspopup 사용 금지 (팝업이 아닌 패널 토글이므로)
  • 정말 랜드마크가 필요할 때만 role="region" + aria-labelledby 고려

키보드 접근성

지원되는 키보드 조작:
  • Enter/Space: 드롭다운 열기/닫기
  • Arrow Down/Up: 옵션 간 이동
  • Home/End: 첫 번째/마지막 옵션으로 이동
  • Escape: 드롭다운 닫기
  • Tab: 다음 포커스 가능 요소로 이동

스크린 리더 지원

스크린 리더 사용자를 위한 기능:
  • 드롭다운 상태 변화 시 자동 알림
  • 선택된 옵션 정보 제공
  • 옵션 개수 및 현재 위치 정보
  • 드롭다운 열림/닫힘 상태 명확한 전달

사용 시 주의사항

개발 시 반드시 확인할 사항:
  • 드롭다운 버튼에 aria-expanded, aria-controls 속성 포함
  • 리스트에 role="listbox" 또는 role="menu" 속성 포함
  • 각 아이템에 role="option" 또는 role="menuitem" 속성 포함
  • JavaScript 초기화 스크립트 포함 (App.a11y.initDropdown)
  • 드롭다운이 닫힌 상태에서 옵션들이 포커스 받지 않도록 설정
  • 외부 클릭 시 드롭다운이 자동으로 닫히도록 구현
권장 사항:
  • aria-haspopup는 스크린리더 사전 안내에 도움이 되지만 심사 기준에 따라 불필요 판단될 수 있으므로 권장(선택) 항목으로 분류합니다.

JavaScript 기능 (업데이트됨)

아래 스크립트는 드롭다운 컴포넌트의 동작을 위해 필요한 기능을 구현합니다. 주요 기능으로는 토글 기능, 항목 선택 기능, 외부 클릭 시 닫기 기능이 포함되어 있습니다.

참고: 아래 스크립트는 드롭다운의 기본적인 동작을 위해 필요합니다. 페이지에 드롭다운 컴포넌트가 포함된 경우 반드시 추가해 주세요.
자동 설정: JavaScript가 자동으로 리스트에 고유 ID를 생성하고 버튼의 aria-controls 속성을 연결합니다. 또한 모든 옵션에 tabindex="-1"을 설정하고 포커스 관리를 담당합니다.
주요 개선사항:
  • 성능 최적화: 윈도우 리사이즈 시 120ms 디바운싱으로 성능 향상
  • 포커스 관리: ESC로 닫기 시 버튼으로 포커스 복귀, 포커스 아웃으로 닫기 시 복귀 안함
  • 모바일 대응: 모바일 환경에서 드롭다운 너비 자동 조정
  • 키보드 접근성: 화살표 키 탐색, Enter/Space 선택, ESC 닫기 지원
  • 스크린 리더 지원: ARIA 속성 자동 관리 및 안정적인 상태 알림
