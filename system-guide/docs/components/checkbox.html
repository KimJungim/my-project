기본 체크박스

단독으로 사용되는 기본 체크박스로, 기본 상태와 비활성화 상태를 지원합니다.

<div class="fd-field">
  <div class="fd-field__ctrl fd-field__ctrl--chk">
    <div class="fd-chk">
      <input id="f-chk-1" type="checkbox" class="fd-chk__el">
      <label for="f-chk-1" class="fd-chk__lbl">체크박스</label>
    </div>
  </div>
</div>

<div class="fd-field">
  <div class="fd-field__ctrl fd-field__ctrl--chk">
    <div class="fd-chk">
      <input id="f-chk-2" type="checkbox" class="fd-chk__el" disabled>
      <label for="f-chk-2" class="fd-chk__lbl">비활성화 체크박스</label>
    </div>
  </div>
</div>

취미 선택 Column(세로)
취미 선택 Row(가로)
참고: 그룹 컨테이너에 .fd-chk-group 클래스를 부여하고, .fd-chk-group--column 또는 .fd-chk-group--row 클래스를 부여하여 세로 또는 가로 배치를 지정합니다.

오류 상태 체크박스

체크박스 오류 발생 시 시각적으로 강조되며, 오류 메시지를 함께 표시해 사용자의 인지를 돕습니다.

약관 동의

*필수 약관에 동의해주세요

<form class="fd-form">
  <div class="fd-field fd-field--error">
    <span id="terms-label" class="fd-field__lbl">약관 동의</span>
    <div class="fd-field__ctrl fd-field__ctrl--chk">
      <div class="fd-chk">
        <input id="terms-1" type="checkbox" class="fd-chk__el" aria-describedby="terms-msg" aria-invalid="true">
        <label for="terms-1" class="fd-chk__lbl">이용약관 동의</label>
      </div>
    </div>
    <!-- 오류 메시지 -->
    <p id="terms-msg" class="fd-field__msg fd-field__msg--error" aria-live="polite">*필수 약관에 동의해주세요</p>
  </div>
</form>
참고(공통):
  • 에러 스타일 기본 규칙: 필드 래퍼(.fd-field)에 .fd-field--error를 토글합니다.
    - 단독 컨트롤로 쓰는 특수 케이스에서만 .fd-field__ctrl--error 사용 가능
  • 보조 정책(선택): 기본은 .fd-field--error입니다. 특수/레거시 케이스에서만 보조 클래스를 사용하세요.
    - .fd-field__ctrl--error (필드 래퍼 없이 단독 컨트롤일 때 한시 적용),
    - .fd-field--chk-error/.fd-field--rdo-error (체크/라디오 전용 레거시 alias).
    - 이들 보조 클래스는 기본 규칙을 대체하지 않으며, 함께 쓰이는 경우 항상 .fd-field--error가 우선합니다.
  • ARIA 적용 대상:
    - 그룹 오류 → 그룹을 fieldset/legend로 구성하는 것을 권장합니다.
    이 경우 fieldset에 aria-describedby로 오류 메시지 id를 연결하고, aria-invalid는 검증 이후 개발단에서 토글합니다.
    - fieldset 사용이 어려운 경우에만 대체로 role="group"/role="radiogroup" + aria-labelledby를 사용합니다.
  • 메시지 연결: 안내/오류 문구에 id를 부여하고 컨트롤(단일) 또는 그룹 컨테이너(예: fieldset 또는 role 컨테이너)에 aria-describedby로 연결합니다. 복수 ID는 공백으로 구분합니다.
  • 초기 상태: 초기 로드 시 aria-invalid는 부여하지 않고, 사용자 입력/검증 이후 상태에 따라 토글합니다.
  • 라디오 필수: 같은 name 집합에서 첫 항목 1개만 required를 둡니다. (중복 지양)
